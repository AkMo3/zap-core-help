Options Active Scan screen

This screen allows you to configure the active scan options:

Száma házigazdák beolvasott egyidejűleg

A maximális számát a házigazdák, hogy a készülék egyidejűleg. Növekvő ezt fel extra megterhelést jelent a számítógép ZAP fut.

Concurrent scanning threads per host

The number of threads the scanner will use per host.
Egyre nagyobb a szálak száma felgyorsítja a vizsgálat, de fel extra megterhelést jelent a számítógép ZAP fut a cél fogadó.

Max results to list

Az eredmények száma jelenik meg, az Aktív Beolvasás lapot.
Megjelenítése egy nagy eredmények száma jelentősen növeli az idő, a vizsgálat tart.

Maximális szabály időtartama (perc, 0 korlátlan)

A maximális idő minden egyes szabály fut a perc. A nulla azt jelenti, hogy nincs határ. Ez lehet használni, hogy megakadályozzák szabályokat figyelembe túlzott mennyiségű idő.

Maximum scan duration (min, 0 is unlimited)

A maximális idő, hogy az egész vizsgálat fut a perc. A nulla azt jelenti, hogy nincs határ. Ezt fel lehet használni annak érdekében, hogy a vizsgálat befejezése körül egy meghatározott ideig.

Késedelem beolvasásakor ezredmásodpercben

A késedelem ezredmásodpercben között minden kérésre.
Setting this to a non zero value will increase the time an active scan takes, but will put less of a strain on the target host.

Adja be a plugin ID fejléc minden aktív szkennelési kéri,

Ha ezt az opciót választja, akkor az aktív szkenner adja be a kérelmet fejléc X-ZAP-Scan-ID with the ID of the scanner that's sending the HTTP requests.

Handle anti CSRF tokens

Ha ezt az opciót választja, akkor az aktív szkenner megpróbálja automatikusan kérés anti CSRF tokens when required.
Vegye figyelembe, hogy ez kísérleti funkció, valamint lassítja a szkennelési folyamat, mivel csak egy szál fogja alkalmazni annak biztosítása érdekében az anti CSRF token kérést nem kap ki a lépés.

A Támadás mód kéri, hogy olvassa újra a csomópontok, amikor hatálya megváltozott

Ha ez az opció van kiválasztva, akkor válassza a Támadás mód kérni fogja, hogy válassza ki, hogy rescan csomópontok hatálya alá.
Ha az opció nincs bejelölve, akkor a következő lehetőség lesz irányítani akár a csomópontok rescanned.

In Attack mode always rescan nodes when scope changed

If this option is selected then when running in Attack mód minden csomópont hatálya lesz rescanned, ha a hatálya változások.
Ez nem javasolt a nagy építkezések, mivel ez sokáig eltarthat.

Alapértelmezett aktív szkennelési politika

The Scan Policy alapértelmezés szerint használt, amikor a start aktív szkennelési.

Attack mode scan policy

The Scan Policy that is used for scanning in Attack mód.

Max progress chart in mins

A maximális idő, a perc, amelyre válaszkódok lesz feltérképezték a Scan Progress dialog.
Tiltsa le a grafikon a lehetőséget meg kell határozni, hogy nulla perc.
