Ang mga diyalogo ng Filter

TANDAAN: Ang mga filter ay may bisa na pinalitan ng Replacer add-on at mga script na mas malakas at may kakayahang umangkop. Ang mga filter ay aalisin sa isang hinaharap na bersyon ng ZAP.

Pinapayagan ka nito na magtakda ng mga filter na inilapat sa mga kahilingan at mga tugon.

Ang mga sumusunod na mga filter ay sinusuportahan ng default.

Baguhin ang user agent sa ibang mga browser

Alamin ang hindi secure o potensyal na nakakahamak na nilalaman sa mga tugon ng HTTP

Alamin at alerto ang 'Set-cookie' sa tugon ng HTTP para sa pagbabago

Iwasan ang cache ng browser (alisin ang KungModifiedSince)

Mag-log ng cookies na ipinadala ng browser

Mag-log natatanging GET query sa file

Mag-log natatanging mga query sa POST sa file

Hiling ng log at tugon sa file

Palitan ang katawan ng kahilingan ng HTTP gamit ang tinukoy na pattern

Palitan ang header ng hiling ng HTTP gamit ang tinukoy na pattern

Palitan ang katawan ng tugon ng HTTP gamit ang tinukoy na pattern

Palitan ang header ng tugon ng HTTP gamit ang natukoy na pattern

Magpadala ng ID ng kahilingan ng ZAP session

Ang filter na ito ay magdaragdag ng isang espesyal na tag ng header sa bawat kahilingan ipadala sa server. Kaya maaari mong subaybayan ang mga kahilingan ng ZAP, kapag sinisiyasat ang iyong sariling web application. Nakatutulong na magkaroon ng isang natatanging id para sa maraming mga kahilingan, kapag nag-parse ng halimbawa ng mga file ng log ng HTTP server.

Ang kahilingan ng header ay binago tulad nito (halimbawa):

X-ZAP-RequestID:  -
  • Ang sessionName ay naka-encode na URL ng pangalan ng session
  • Ang numero ay nagsisimula sa pamamagitan ng isa; nadagdagan sa bawat kahilingan; ay may-bisa lamang sa bawat sesyon
