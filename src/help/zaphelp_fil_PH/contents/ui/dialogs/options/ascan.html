Aktibong Opsyon ng Scan screen

Ang screen na ito ay pinahihintulutan ka s configure ang aktibong scan ng opsyon:

Bilang ng sabay sabay na mga host na na-scan

Ang pinakamataas na numero ng mga host na nascan na sa parehong oras. Ang pagtaas na ito ay maaaring maglagay ng karagdagang strain sa computer na ZAP ay tumatakbo.

Sabay sabay na pagiskan sa mga thread kada host

Ang bilang ng mga thread ay iiskanin ang gumagamit kada host.
Ang pagdagdag ng bilang ng mga thread ay mas mapapabilis ang iskan pero ito ay maglalagay ng karagdagang hirap sa kompyuter ng ZAP ay tumatakbo at ang mga tinutukoy na host.

Mataas na mga resulta para sa listahan

Ang bilang ng resulta na makikita sa aktibong Scan tab.
Ang pagpapakita ng malaking bilang ng resulta ay maaaring higit na mapataas ang oras ng pagtagal sa pagiskan.

Pinakamataas na tagal ng patakaran (minuto, 0 ay walang limitado)

Ang pinakamataas na oras ay pwedeng indibidwal na patakaran ay maaaring paganahin ng ilang minuto. Zero ibigsabihin ay walang limit. Ito ay maaaring gamitin para maiwasan ang mga patakaran na kumukuha ng malaking halaga ng oras.

Pinakamataas na tagal ng pagiskan (minuto, 0 ay walang limitado)

Ang pinakamataas na oras na ang buong iskan ay maaaring paganahin ng ilang minuto. Zero ibig sabihin ay walang limit. Ito ay maaaring gamitin para matiyak na ang isang iskan ay kompleto na sa loob ng itinakdang oras.

Ang pagkaantala kapay nagiiskan sa milliseconds

Ang pagkaantala sa mga millisecond sa pagitan ng bawat aplikasyon.
Ang pagtatakda nito sa hindi serong halaga ay makadaragdag sa oras na pagkuwa sa Aktibong iskan, pero ito ay may mas mababang pwersa sa pinupuntong host.

Iniksyon ng plugin ID sa uluhan para sa lahat ng mga kahilingan sa aktibong scan

Kung ang opsyon na ito ay napili ang aktibong iskaner ay iinhiksyon ang aplikasyon sa ulohan X-ZAP-Scan-ID kasama ang ID ng iskaner na magpapadala ng mga kahilingang HTTP.

Hawakan ang anti CSRF ng mga token

Kung ang opsyon na ito ay napili kung magkagayon ang aktibong iskaner ay mapipilitan na maging awtomatikong kahilingan anti CSRF kailan kailangan ang mga token.
Paalala na ang ekspirementong ito ay pagpapagana at pag bagal ng proseso sa iskaning bilang isang thread ay maaaring gamitin para masigurado na ang anti CSRF token request ay hindi mawawala sa paghakbang.

Sa mode ng attack prompt muling i-iiskan ang mga node pagka ang saklaw ay nagbago

Kung ang opsyon na ito ay napili kung magkagayon ikaw ay pumili ng attack mode ikaw ay maaaring magprompt para pumili kung muling i-iskan ang mga node sa scope.
Kung ang opsyon ay hindi napili kung magkagayon ang mga sumusunod na opsyon ay makokontrol kung ang mga node ay muling iiiskan.

Sa attack mode laging muling iiskan ang mga node kung ang scope ay nagiba

Kung ang opsyon na ito ay napili kung magkagayon kung gagana sa attack mode ang lahat ng mga node sa scope ay maaaring iiskan muli kung ang scope ay nagpalit.
Ito ay hindi nirerekomenda para sa malaking mga site bilang ito ay pwedeng sumakop ng mahabang oras.

I-default ang patakaran sa aktibong iskan

Ang Patakaran ng iskan na ginamit sa pamamagitan ng default kung ikaw ay magsisimula sa aktibong iskan.

Ang patakaran sa attack mode

Ang Patakaran ng Iskan na ito ay ginamit para sa pagiskan sa attack mode.

Mataas na progresong tsart sa mga minuto

Ang pinakamataas na oras sa minuto na kung saan ay para sa tugon ng mga code ay maaaring maitsart sa Isinasagawa dialog ng scan.
Para wag paganahin ang tsart ang opsyon ay dapat itakda sa serong minuto.
