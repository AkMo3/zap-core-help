Inilabas na 2.5.0

Ang mga sumusunod na pagbabago ay ginawa sa inilabas na ito:

Mga pagbabago sa ZAP API:

VIEW authorization / getAuthorizationDetectionMethod

Nagbabalik ngayun ang datos na nakabalot sa isang bagay na tinatawag na authorizationDetectionMethod: 
{"authorizationDetectionMethod":{"statusCode":"-1"..."headerRegex":""}}
maliban sa: 
{"statusCode":"-1"..."headerRegex":""}

Tingnan ang context / context + technologyList

Ngayon, ibalik ang parehas na datos na nakabalot sa bagay na tinatawag na "context": 
{"context":{"id":"1", ..., "inScope":"true","loggedOutPattern":""}}
maliban sa: 
{"id":"1", ..., "inScope":"true","loggedOutPattern":""}

ACTION spider / scan + scanAsUser

Ngayon, ang parehas na pagsuporta sa bagong opsyunal 'subtreeOnly' parameter na syang nagbibigay hangganan sa spider hanggang sa tinututkoy na substree. Ang 'url' parameter ay isa na ngayong opsyunal, hanggat ang mabisang 'context' parameter ay natutustusan

Ang bagong bumubuo sa 'stats'

Ang bagong bumubuo ng API 'stats' ay nagbibigay daan sa stats na ngayon ay pinanatili ng ZAP.

Tandaan na ang mga nananatiling mga bahagi ay dapat magkaroon din ng mga bagong pagpapatakbo, tingnan ang API Web UI para sa iba pang mga detalye.
