Ang mga sesyon ng HTTP

Ang tool na ito ay patuloy na sinusubaybayan ang mga gumaganang sesyon ng HTTP sa patikular na site at pahintulutan ang Zaproxy user na puwersahin ang lahat ng mga request na mapunta sa isang partikular na sesyon. Talagang, pinahihintulutan nita ang gumagamit na mapadali ang paglipat lipat sa pagitan ng gumagamit ng sesyon sa isang Site at sa paggawa ng bagong sesyon ng hindi "nasisira" ang mga gumagana na.

Ito ay nakabatay sa konsepto sa mga session Token, kung saan ang mga HTTP message parameter (para sa mga bagong Cookies lamang) kung saan pinahihintulutan ang HTTP server na komunekta sa request message ng anumang nakalipas na request o data stored. Sa kaso ng Zaproxy, conceptually, ang mga sesyon token ay nauuri sa dalawang kategorya: ang default session token at ang site session token. Ang default session token ang isa sa mga pwedeng ilagay ng gumagamit sa mga Option screen at ang token na nga ay automatikong kinikilala ng kahit anong site sa pamamagitan ng default session token ( eg. phpsessid, jsessionid, etc). Ang mga site session token ay ang set ng mga token para sa isang partikular na site at karaniwang na seset up gamit ang mga popup menu na available lamang sa Params Tab.

Ang tool na ito ay automatikong na dedetek, gamit ang mga defined session token o ng mga automatikong detected default session token, ng kahit anong HTTP session na umaiiral sa komunikasyon. Ang mga detected session ay makikita sa HTTP Session Tab.

Ang gumagamit maaring, gamitin ang button na available sa mga HTTP Session Tab, na gumawa ng bagong session ng hindi nasisira ang umiiral na o sapilitang paganahin ang mga session. Kapag ang isang sesyon ay 'aktibo', ang lahat ng mga outbound request na ipinadala nito sa corresponding Site ay binabago, ang mga session token ay upang tumugma sa aktibong sesyon. Sa parang ito, ang gumagamit ay madaling mapupuwersa ang ilang mensahe na maging 'parte ng' isang partikular ng sesyon at lumipat sa pagpapadala ng mensahe sa isa pang sesyon.

Ang HTTP Session tool ay isinasaayos gamit ang mga HTTP Session option screen.
