リリース 1.0.0

本リリースでは以下のような変更が行われました。

重要な変更

ヘルプ画面

ZAPが提供する全ての機能について、ヘルプ画面が追加されました。
F1 キーを押すと選択しているタブに関連するヘルプ画面が表示されるよう、状況依存型ヘルプも実装されました。

ブレーク

The implementation of ブレーク has been changed to be closer to that provided by software Integrated Development Environments.
The break point controls have been moved to the new トップレベルツールバー.
ブレークポイントは次のツリー上で右クリックすることで設定することもできます。 サイト 及び 履歴タブ.

レポートメニュー

　 レポートメニュー は大幅に変更され、次のメニューアイテムを含むようになりました。
    レポートを生成する.今すぐレポートを生成する。
    メッセージをファイルにエクスポート...[ファイル] メニュー から移動しました。
    レスポンスをファイルにエクスポート...[ファイル] メニュー から移動しました。
    全てのURLをファイルにエクスポート...新機能
    他セッションとの比較...新機能

履歴検索の改善

The new 検索タブ allows you to search for regular expressions in all of the URLs, requests and responses.
Next and previous buttons allow you view the found search terms in the リクエスト 及び Response tabs.

履歴フィルタ

履歴フィルタ ツールバー has been added which allows you to restrict which requests are displayed.
Requests can be filtered based on the HTTP methods, the HTTP result codes, and any タグ, アラート, or ノート that have been applied.

ノート

You can now associate ノート with any request.

Multiple Tags

You can now associate multiple タグ with any request.

More control over the alerts

　 アラートダイアログを追加 allows you to manually add or change an alert.

新しいエンコーダー/デコーダー

A new Encode/Decode/Hash dialog has replaced the previous Encoder/Decoder.

静的スキャン

静的スキャン が追加されました。
In this release ZAP will only use passive scanning for automatically adding タグ.
In future releases it may also raise アラート for potential issues.

小変更：

トップレベルツールバー

トップレベルツールバー has been added which provides a set up buttons for commonly used functionality.

アラートフッターの概要

Counts of the High, Medium, Low and Informational アラート are now displayed in the フッター.

新しい見た目と操作感

新しい見た目と操作感の「Nimubus」が使用されています。

プロキシ パスワードの扱い

The Options Connection screen 新たに 「起動時にプロキシの認証情報の入力を求める」オプションを備えます。
本オプションが選択されている場合、プロキシのパスワードはハードディスク上に保存されません。

サイトタブからの再送信

　 サイトタブ has a new right click menu item that allows you to resend the request after making any changes to it that you want to.

User directory maintained between sessions

The last directory used by the user is now stored in the config file so that it is maintained between sessions.
