Sürüm 2.6.0

Bu sürüm, özellikle ZAP API'sinde çok sayıda değişiklik içeriyor.
ZAP'ı API aracılığıyla tamamen kontrol edilebilir hale getirme hedefimize yönelik olarak, önemli sayıda yeni API son nokta ekledik. Ayrıca mevcut bitiş noktalarının bazılarını değiştirdik ve her durumda bu değişiklikler geriye dönmeye uyumludur.

Tam sürüm, ayrıca çeşitli tarayıcılar aracılığı ile ZAP etkileşimini daha da kolaylaştırmak amacıyla platforma özgü ağ sürücüsü ile birlikte yeni bir JxTarayıcı eklentisi içerir.

API güvenlik değişiklikleri

hata ödenek programı ile bize bildirilen konular karşısında APİ güvenliğini değiştirdik. Güvenlik Açıkları ayrıntıları aşağıda belirtilmiştir.
Güvenli ortamda ZAP kullanırsanız, devre dışı bırakmanız için seçenek eklememize rağmen, güvenlik değişiklikleri zorunlu olarak geriye dönük uyumlu değildir.

Varsayılan olarak, tüm API çağrıları artık API anahtarını veya bir nonce'ı gerektirir. Bunlar, URL parametreleri, POST parametreleri veya başlıklar aracılığıyla sağlanabilir. Desteklenen ZAP API istemcileri (Java ve Python da dahil olmak üzere) bir üstbilgi aracılığıyla API anahtarını sağlamak üzere güncellendi. Nonces, ZAP tarafından üretilir ve ZAP API'sine erişmesi gereken ZAP eklentileri tarafından kullanılmak üzere tasarlanmıştır. Tüm ayrıntılar için APİ ekran seçenekleri bakınız.

Bunlar güvenlik ile ilgili yeni APİ seçenekleri serisidir:

  • UI Etkin - Etkinleştirildiğinde, API Web Kullanıcı Arayüzü, ZAP'ı proxy olarak kullanabilen tüm makinelerde kullanılabilir. Bu varsayılan olarak etkindir.
  • API kullanımına izin verilen IP adresleri - Varsayılan olarak yalnızca ZAP çalışan makine ZAP API'sine erişebilir. Uygun regex kalıpları ekleyerek diğer makinelere API'ye erişime izin verebilirsiniz. Sadece güvendiğiniz İP adreslerini eklemelisiniz.
  • Güvenli işlemler için bir API anahtarı gerektirmeyin - Etkinleştirildiğinde, API anahtarı, Görünümler veya 'güvenli' olarak kabul edilen diğer işlemler için gerekli değildir, başka bir deyişle ZAP'ta herhangi bir değişiklik yapmayan işlemler. Bu tür işlemler, uyarı, mesajlar ve dosya sistemi yolları gibi ZAP verilerine erişebilirler. ZAP'ın varlığını saptamak için web uygulamaları tarafından da kullanılabilirler.
  • İzin hatalarını API ile bildirin - Etkinleştirilirse, ZAP, API aracılığıyla izin hataları bildirir; bu, ZAP'ın varlığını tespit etmek için web uygulamaları tarafından kullanılabilir. Bu güvenli bir ortamda ciddi bir sorun değildir, ancak potansiyel olarak kötü amaçlı sitelere karşı ZAP kullanıyorsanız, etkinleştirmemelisiniz.
Bütün ZAP ayarları komut satırıyla ZAP'ı başlattığında özel olabilir - bkz SSS(Sıkça Sorulan Sorular) API Anahtarı tüm detayları için.
Güçlü bir İçerik Güvenlik Politikası ile birlikte API'ye çok daha fazla güvenlik üstbilgisi ekledik.

ZAP API Değiştirilmiş Bitiş Noktaları:

EYLEM ascan / scan

Url parametresi isteğe bağlıdır ve isteğe bağlı bir contextId parametresi eklenmiştir. Bunlardan birini sağlamanız gerekir.

EYLEM ascan / scanAsUser

Url ve contextId parametreleri artık opsiyoneldir. Bunlardan birini sağlamanız gerekir.

EYLEM ascan / addScanPolicy

İsteğe bağlı alertThreshold ve attackStrength parametreleri eklendi.

ZAP API Yeni Bitiş Noktaları:

GÖRÜNÜM ascan / optionMaxRuleDurationInMins

Sıfırın sınırsız olduğu bir tarama kuralının çalışabileceği maksimum süreyi dakika cinsinden verir.

GÖRÜNÜM ascan / optionMaxScanDurationInMins

Tam bir taramanın çalışabileceği maksimum süreyi dakika cinsinden döndürür, sıfır sınırsızdır.

EYLEM ascan / setOptionMaxRuleDurationInMins

Bir tarama kuralının çalışabileceği maksimum süreyi dakika cinsinden ayarlar, sıfır sonsuzdur.

EYLEM ascan / setOptionMaxScanDurationInMins

Bütün taramanın çalıştırabileceği maksimum süreyi dakika cinsinden ayarlar, sıfır sonsuzdur.

EYLEM ascan / updateScanPolicy

Belirtilen tarama ilkesini alertThreshold veya attackStrength ile günceller.

Görünüm Sonu / isBreakAll

Eğer ZAP, hem istekleri hem de yanıtları keserse Doğru'ya döner.

Görünüm Sonu / isBreakRequest

Eğer ZAP taleplerde kesintiye uğrarsa Doğru'ya döner.

Görünüm sonu / isBreakResponse

Eğer ZAP cevapları keserse doğruya döner.

Görünüm Sonu / httpMessage

Halihazırda engellenen HTTP iletisini döndürür (varsa).

EYLEM kesme / kesme

Küresel ara işlevselliğini kontrol eder. Türü aşağıdakilerden biri olabilir: http-hepsi http-isteği veya http-yanıtı. Durum (Belirtilen türde arayı açmak için) doğru veya (arayı kapatmak için) yanlış olabilir. Faaliyet alani şu anda kullanılmıyor.

EYLEM sonu / setHttpMessage

Sağlanan verileri şu anda engellenen mesajın üzerine yazar.

EYLEM kesme / devam etme

Halihazırda engellenen iletiyi gönderir ve genel istek/yanıt kesme noktalarını sıfırlar.

EYLEM kesme / girme

Halihazırda engellenen iletiyi gönderir, bir sonraki istek veya cevap otomatik olarak engellenir.

EYLEM kesme / düşme

Halihazırda engellenen mesajı bırakır.

Çekirdeği Görüntüle / optionDnsTtlSuccessfulQueries

Başarılı DNS sorgularının TTL'lerini (saniyeler içinde) getirir.

EYLEM çekirdek / Istekgonderme

Aşağıdaki yeniden yönlendirmelerle, isteğe bağlı olarak HTTP isteği gönderir. Eğer varsa, gönderilen istek ve alınan yanıt ve takip edilen yönlendirmeleri döndürür. İstek yollandığında (takip eden yeniden yönlendirmeyle) mod zorla kabul ettirilir, özel ayarlanan isteklere, kapsam dışındaysa 'Güvenli' modda veya ' Koruma' modunda izin verilmez.

EYLEM çekirdek / setOptionDnsTtlSuccessfulQueries

Başarılı DNS sorgularının TTL'sını (saniyeler içinde) ayarlar (ZAP yeniden başlatıldıktan sonra).

Diğer çekirdek / mdreport

Markdown formatında bir rapor oluşturur.

Görüntüle httpSessions / sites

Oturumu olan bütün siteleri getirir.

INCELE örümcek / butunUrl

Anti CSRF Test Formu ve API Anahtar Açıklaması ile RCE

Eğer kullanıcı Anti CSRF Sinama Formu'nu, özel olarak hazırlanmış bir HTML sayfasına karşı kullanırsa, API anahtarı bu siteye sızdırılır. Site daha sonrasında ZAP APİ'e ulaşabilir ve ZAP komut dosyalarını yüklemek dahil bütün eylemleri gerçekleştirebilir. Komut dosyaları sadece 'yerel' dosya sistemlerinden yüklenebilir, ancak kullanıcı Windows'ta ZAP çalıştırıyorsa, saldırgan kamuya açık bir SMB paylaşımı yoluyla kötü amaçlı bir komut dosyası hazırlayabilir. Yerel bir dosya olmak için ZAP'a görünür ve bu nedenle komut dosyası APİ tarafından yükelenebilir ve çalıştırılabilir.
Değişken eklentilerde APİ anahtarının sızmasını önlemek amacıyla tek seferlik anahtar kullanımının olduğu gibi APİ anahtarı gereksinimi veya APİ operasyonlarında kullanmak üzere tek seferlik anahtar gereksinimi bu güvenlik açığının doğrudan bir sonucudur.

2.5.0 sürümüne kadarki (2.5.0 sürümü dahil) bütün versiyonlar için ZAP Windows Yükleyicisi'nin, Windows 7 ve daha önceki sürümlerindeki DLL Korsanlığına, güvenlik açığı vardır. Bu, 3. parti İnnoKurulum yükleyicide güvenlik açığına sahiptir. Artık 2.6.0 Yükleyiciler (bütün platformlarda) 4JYükle kullanılarak oluşturulur.

Herhangi bir nedenle Windows 7'ye veya daha eski sürümlerine ZAP'ın önceki sürümlerini yüklemek durumunda kalırsanız, kurucuyu çalıştırmadan önce temiz bir dizine taşımanızı öneririz.

Burp Suite'in Install4J'yi kullandığını ve böylece Install4j-üretilen yükleyicilerde gelecekteki güvenlik açıklarının Burp Suite hata ödül programı için geçerli olabileceğini unutmayın: https://hackerone.com/portswigger

Jenerik: James Kettle (Burp Suite)

Başlangıç Uygulama Parametre Ekleme yoluyla Rastgele Kod Çalıştırma

Eğer kullanıcı hedeflenen uygulamayı ZAP lı parametre içermesini isteyerek başlatırsa, HTML parametreleri özellikle rastgele kod çalıştırmaya neden olacak şekilde yaratılabilir.
Eklenti Başlangıç Uygulamaları bu sorunları gidermek için güncellendi - Eklentiyi kullanmaya devam etmeden önce bütün ZAP kullanıcıları bu yeni versiyonu yüklemelidir.

Jenerik: Artemy Bogdanov (@Abr1k0s)

Anti CSRF Test Formu üzerinden XSS

CSRF önleyici deneme formunun, özel olarak hazırlanmış HTML sayfasına karşı çalıştırılırsa, XSS saldırılarla karşı güvenlik açığı vardır.
APİ artık bu tarz sorunları önlemek amacıyla güçlü İçerik Güvenlik İlkesi'ni kullanır

Kredi: g_sato - https://bugcrowd.com/g_sato

DNS Rebinding'e APİ'nin Güvenlik Açığı

APİ'nin DNS Rebinding saldırılarına karşı güvenlik açığı vardır. Şu anda sunucu bağlantıyı denetlemektedir ve beklenmedik sunuculardan olan bütün istekleri geri çevirir.

Kredi: Artemy Bogdanov (@Abr1k0s)
