Aba de histórico

A aba de histórico mostra uma lista de todas as solicitações/requests, na ordem em que elas foram feitas.
Para cada solicitação, você pode ver:
     O índice de solicitação - cada solicitação é numerada, começando em 1
     O método HTML, por exemplo, GET ou POST
     A URL solicitada
     O código de resposta HTTP
     Um breve resumo do que significa o código de resposta HTTP
     O tempo que levou a toda solicitação
     Alertas sobre o pedido
     Quaisquer notas que você tiver adicionado ao request
     Qualquer Tag na solicitação
Selecionar um pedido irá exibi-lo na Aba de Request e Aba de Resposta acima.

Barra de ferramentas de filtro

Uma barra de ferramentas filtro é fornecida para restringir quais solicitações serão exibidas.
Clicar sobre o botão de Filtro exibe o diálogo do histórico de filtro.
Um resumo do filtro usado naquele momento é exibido à direita do botão.

Menu do botão direito

Clicar com o botão direito em um nó faz aparecer um menu que lhe permitirá:

Ataque

O menu de ataque tem os seguintes submenus:

Varredura ativa...

Isto irá lançar o diálogo de Varredura Ativa que permite a você iniciar uma Varredura Ativa no ponto de partida definido para a solicitação que você escolheu.

Spider...

Isto irá lançar o diálogo de Spider que permite a você iniciar o Spider no ponto de partida definido para a solicitação que você escolheu.

Incluir no contexto

Este menu permite que você inclua os nós selecionados e todos os seus subordinados no contexto especificado.
Você também tem a opção para criar um novo contexto.
O diálogo Contextos da sessão será exibido para que você possa fazer alterações adicionais.

Excluir do contexto

Este menu permite que você exclua os nós selecionados e todos os seus subordinados do contexto especificado.
Os diálogos de Contextos da sessão serão exibidos para que você possa fazer alterações adicionais.

Flag como contexto

Este menu tem os seguintes submenus para cada um dos contextos que você definiu:

Context name Form-based Auth Login request

This identifies the specified node as a login request for the specified context.
Você pode ter apenas um nó identificado como tal em cada contexto.
O diálogo de Session Context Authentication screen will be displayed to allow you to make any additional changes.

Context name Data driven node

This identifies the specified node as Data driven content for the specified context.
O diálogo de Session Context Structure screen will be displayed to allow you to make any additional changes.

Excluir de

Este menu possui os seguintes submenus:

Proxy

Isto irá excluir do proxy os nós selecionados. Eles ainda passarão pelo ZAP, mas não serão exibidos em qualquer uma das guias.
Isto pode ser usado para ignorar URLs que você sabe que não são relevantes para o sistema que você está testando.
Os nós podem ser incluídos novamente através do diálogo Propriedades da Sessão

Scanner

Isto irá impedir que os nós selecionados entrem na varredura ativa.
Os nós podem ser incluídos novamente através do diálogo Propriedades da Sessão

Spider

Isto irá impedir que os nós selecionados sejam percorridos pelo spider.
Os nós podem ser incluídos novamente através do diálogo Session Properties

Gerenciar Tags...

Isso irá abrir o diálogo Manage Tags dialog que permite a você alterar as tags associadas com a solicitação.

Notas...

Isso irá abrir o diálogo Add Note dialog que permite a você gravar notes relacionadas com o pedido.

Deletar

Isto irá remover o nó e todos os seus 'filhos' do ZAP.
No entanto, eles podem ser adicionados de volta, para impedir que isso aconteça no menu 'Excluir de'.

Pausa...

Isso irá abrir o diálogo Add Break Point dialog que permite a você definir um ponto de interrupção naquela URL.

Escanear este histórico

Isso executará uma active scan sobre a URL relacionada ao request selecionado.

Alertas para este alvo

Se a URL selecionada tem alerts associadas a ela, então eles serão listados sob este menu.
Selecionar um dos alertas fará com que ele seja exibido.

Reenviar...

Isso irá abrir o Resend dialog que lhe permite reenviar o request depois de fazer quaisquer alterações que você deseje.

Novo Alerta...

Isso irá abrir o Add Alert dialog que permite a você registrar manualmente um novo alert para este request.

Mostrar na Aba de Sites

Isto irá mostrar a mensagem selecionada na Sites tab.

Ver no Navegador

Isto irá abrir a URL do nó selecionado em seu navegador padrão.

Gerar formulário de teste anti CSRF

Isto irá abrir a URL de um formulário destinado à detecção de problemas de CSRF.
Ele só vai ser habilitado para solicitações POST - caso o API esteja habilitado e se o Java suportar a abertura de URLs em um navegador na sua plataforma.
