Tela de autenticação de contexto de sessão

Esta é uma das Telas de Contexto de Sessão que permite a você gerenciar o modo pelo qual a Autenticação está sendo feita para o contexto. Note-se que mudar o método de autenticação depois que os usuários foram definidos fará com que eles sejam deletados, porque o tipo de credencial do usuário precisa coincidir com o esquema de autenticação. Depois de selecionar o tipo de método de autenticação, as opções que precisam ser configuradas dependem do método de autenticação.

Autenticação manual

Nenhuma configuração é necessária para este método de autenticação. Leia mais...

Autenticação baseada em formulário

Para configurar este método de autenticação, você precisa fornecer a url de login, para a qual a requisição de login é enviada, o corpo da solicitação/requisição (Dados do POST), se necessário e identificar os parâmetros usados no fornecimento do 'username' e 'senha'. Não se for fornecido nenhum corpo de solicitação (request), a solicitação de logon é executada como um HTTP GET, caso contrário será usado um HTTP POST. As credenciais na verdade são configuradas na aba de usuários. Leia mais...

Autenticação HTTP/NTLM

Para configurar este método de autenticação, você precisa fornecer o hostname e a porta do servidor onde a autenticação é feita e o alcance que essas credenciais têm. Elas são configuradas na aba de usuários. Leia mais...

Autenticação baseada em script

Para usar esse método de autenticação, você primeiro precisa escrever (e salvar) um Script de autenticação usando a Aba de scripts (consulte os exemplos fornecidos e templates para este tipo de script na aba Scripts). Em seguida, você precisa fornecer o nome do script na lista drop down. Depois de selecionar o script, você precisa pressionar o botão Carregar, para carregar todos os requisitos do script. Quaisquer parâmetros que você especificou, tais como Obrigatório ou opcional no script serão mostrados na interface a ser definida. Os valores desses parâmetros estarão disponíveis para serem usados no script, durante a autenticação, como visto nos exemplos fornecidos de Scripts de autenticação. Depois de fazer quaisquer alterações nos parâmetros exigidos pelo script de autenticação você terá de recarregá-lo. Caso contrário, os parâmetros mostrados na interface podem não ser aqueles usados na autenticação, e assim erros poderão ocorrer. As credenciais utilizadas para cada usuário durante o processo de autenticação também podem ser especificadas no Script de autenticação e ser configuradas na aba de usuários. Leia mais...
