Diálogos de contextos de sessão

Estas telas permitem que você gerencie contextos.

Há um conjunto de telas para cada contexto que você definir.

Parte superior

Isso permite que você defina o nome do contexto e a sua descrição.

Incluir no contexto

Isso permite que você gerencie as URLs que serão incluídas no contexto.
URLs que não correspondem a nenhum do regexs não serão incluídas no contexto.

Excluir do contexto

Isso permite que você gerencie as URLs que serão excluídas do contexto.
Você só precisa especificar regexs para URLs que você não deseja incluir, mas que combinem com uma ou mais das regexes de 'incluir'.

Tecnologia

Isso permite que você especifique as tecnologias utilizadas no contexto, se conhecidas.
Por padrão, todas as tecnologias são incluídas.
Se você excluir tecnologias que você sabe que não são usadas, então isto pode acelerar a varredura ativa já que as regras específicas para as tecnologias excluídas podem ser ignoradas.

Autenticação

Isso permite a você gerenciar a maneira pela qual a Autenticação está sendo feita para o contexto. Note-se que mudar o método de autenticação depois que os usuários foram definidos fará com que eles sejam deletados, porque o tipo de credencial do usuário precisa coincidir com o esquema de autenticação. Depois de selecionar o tipo de método de autenticação, as opções que precisam ser configuradas dependem do método de autenticação.

Autenticação manual

Nenhuma configuração é necessária para este método de autenticação. Leia mais...

Autenticação baseada em formulário

Para configurar este método de autenticação, você precisa fornecer a url de login, para a qual a requisição de login é enviada, o corpo da solicitação/requisição (Dados do POST), se necessário e identificar os parâmetros usados para fornecer o 'username' e 'senha'. Não se for fornecido nenhum corpo de solicitação (request), a solicitação de logon é executada como um HTTP GET, caso contrário será usado um HTTP POST. As credenciais originais são configuradas na Aba de usuários . Leia mais...

Autenticação HTTP/NTLM

Para configurar este método de autenticação, você precisa fornecer o hostname e a porta do servidor onde a autenticação é feita e o alcance que essas credenciais têm. Elas são configuradas na Aba de Usuários . Leia mais...

Autenticação baseada em script

Para usar esse método de autenticação, você primeiro precisa escrever (e salvar) um Script de autenticação usando a Aba de scripts (consulte os exemplos fornecidos e modelos para este tipo de script na aba Scripts). Em seguida, você precisa fornecer o nome do script na lista suspensa. Depois de selecionar o script, você precisa pressionar o botão Carregar, para carregar todos os requisitos do script. Quaisquer parâmetros que você especificou, tais como obrigatório ou opcional no script serão mostrados na interface a ser definida. Esses valores estarão disponíveis para serem usados no script, durante a autenticação, como visto nos exemplos fornecidos de Scripts de autenticação. Depois de fazer quaisquer alterações nos parâmetros exigidos pelo script de autenticação você terá de recarregá-lo. Caso contrário, os parâmetros mostrados na interface podem não ser aqueles usados na autenticação, e assim erros poderão ocorrer. As credenciais utilizadas para cada usuário durante o processo de autenticação também podem ser especificadas no Script de autenticação e ser configuradas na Aba de usuários. Leia mais...

Gerenciamento de sessão

Isso permite a você gerenciar a maneira pela qual o gerenciamento de sessão está sendo feito para o contexto. Depois de selecionar o tipo de método de gerenciamento de sessão, as opções que precisam ser configuradas dependem desse método de gerenciamento de sessão.

Gerenciamento de sessão baseado em cookies

Nenhuma configuração é necessária para este método de gerenciamento de sessão. Leia mais...

Usuários

Isso permite a você configurar um conjunto de usuários que podem ser utilizados para várias outras ações em todo o aplicativo.

A seção de credenciais dos usuários depende do método de autenticação configurado para o contexto.

Regex de URL

Nos painéis Incluir em *, Excluir de * e nos indicadores de login/logout do painel de Autenticação , você pode digitar expressões regulares para definir URLs excluídas. Você pode dar escape em determinados metacaracteres com uma barra invertida, e também pode usar sequências \Q...\E Todos os caracteres entre o \Q e o \E são interpretados como caracteres literais. Por exemplo, o \Q*\d+*\E coincide com o texto literal * \d+*. Esta sequência de escape é usada no ZAP quando você exclui URLs através de alguns menus de contexto.
Nota: Se sua URL contém um "\E", então você tem que dar os seguintes passos ao usar a sequência de escape \Q...\E :
  • Abra a seqüência de escape
  • Feche a seqüência de escape antes do "caractere" \E
  • Escape a barra invertida
  • Abra depois do "\E" outra seqüência de escape;
  • Feche a seqüência de escape como normalmente faria.

Exemplo: subdomain.example.com/path?a=\E&moredata=2 deve aparecer como \Qsubdomain.example.com/path?a=\E\\E\Q&moredata=2\E
