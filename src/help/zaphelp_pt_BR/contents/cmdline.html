Linha de comando

O ZAP suporta as seguintes opções em linha de comando:
    -cmdRoda o ZAP 'inline', ou seja, sem a interface do usuário nem daemon
    -configSubstitui o par 'variável=valor' no arquivo de configuração
    -daemonO ZAP é iniciado no modo de 'daemon', ou seja, sem interface de usuário
    -dirUsa o diretório especificado no lugar do padrão
    -installdirSubstitui o código que detecta onde ZAP foi instalado pelo diretório agora especificado em -installdir
    -hMostra todas as opções de linha de comando disponíveis, incluindo aquelas adicionadas pelos Complementos
    -helpO mesmo que -h
    -hostSubstitui o host especificado no arquivo de configuração para ser usado como proxy
    -portSubstitui a porta do proxy especificado no arquivo de configuração
    -versionInforma a versão do ZAP
    -newsessionCreates a new session at the given location
    -sessionOpens the given session after starting ZAP

As opções '-sessão ' e '-newsession' são mutuamente exclusivas - usa-se uma ou a outra. Se você tentar usar as duas surgirá um aviso de erro e o ZAP será encerrado (se não estiver em GUI - interface gráfica).
Relative paths to session file are resolved against the "session" directory located in ZAP's home directory (default or specified with -dir option).
Configuration keys should be specified using the dot notation based their location in the XML of the configuration file, eg:
-config api.key=12345 -config connection.timeoutInSecs=60
Note that add-ons can add extra command line options.

Exemplos:

  • Inicie o ZAP no modo 'daemon' com uma nova sessão criada em um determinado caminho assim: 
    -daemon -newsession session
  • Crie um relatório da última varredura de uma sessão existente e encerre o ZAP em seguida: 
    -last_scan_report /path/to/save/report.xml -session /path/to/existing/session -cmd
