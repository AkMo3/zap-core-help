Usuários

Os usuários são as representações de ZAP para os usuários dos sites/webapps. Eles permitem que determinadas ações sejam executada a partir do ponto de vista de um usuário da aplicação. Para cada contexto, um conjunto de usuários pode ser definido, podendo ser usado em ações relacionadas ao contexto. Most commonly, during various scans the request messages can be sent from the point of view of a User.

O conceito de usuário é bem ligado aos conceitos de autenticação e Gerenciamento de sessão. Quando um usuário é usado pela primeira vez em algum lugar no ZAP, uma autenticação é realizada (de acordo com o método de autenticação definido para o contexto) e uma sessão é criada e configurada para este usuário (de acordo com o gerenciamento de sessão definido para o contexto). After that, requests sent from the point of view of a User are modified (if necessary) and sent in such a way that the web server identifies them as being sent by an authenticated webapp/website user. Se uma mensagem é enviada da perspectiva de um usuário e a resposta recebida parece não autenticada (conforme identificado pelos indicadores de autenticação logado e deslogado), uma nova autenticação é executada e a sessão é atualizada.

In order to perform the authentication of a user on a website / in a webapp, the Authentication Method defines how the authentication is done (the process), while the necessary credentials (the exact identifiers) are dependent on the user, so, in ZAP, they are configured in the Users.
