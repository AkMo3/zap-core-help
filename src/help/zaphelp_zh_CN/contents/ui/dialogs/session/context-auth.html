会话上下文身份验证屏幕

这是其中一个 会话上下文屏幕 它将允许您以此方式来管理 身份验证 正在为上下文进行操作。 请注意，在用户定义之后更改身份验证方法，这将导致被删除，因为用户凭据类型需要与身份验证方案匹配。 选择身份验证方法类型之后，需要配置的选项取决于身份验证方法。

手动身份验证

此身份验证方法不需要配置。 读取 更多...

基于表单的身份验证

为了配置此身份验证方法，您需要提供 登录url, to which the login request is performed, the 请求主体 (POST数据)，如果需要，并识别 参数 用于提供的"用户名"和"密码"。 如果未提供请求正文，则将以HTTP GET的方式执行登录请求，否则将使用HTTP POST。 The credentials themselves are configured in the 用户 标签。 读取 更多...

HTTP/NTLM身份验证

为了配置此身份验证方法，您应该提供 主机名称 and the 端口 服务器的身份验证已完成和 realm 凭据适用于。凭据本身配置在 用户 标签。 读取 更多...

基于脚本的身份验证

为了使用此身份验证方法，您 第一 需要写(并保存) 一个 认证脚本 使用 脚本 选项卡(请参阅脚本选项卡中的此脚本类型提供的示例和模版)。 然后您需要在下拉列表中提供脚本名称。 选择脚本后，您需要按 加载 按钮，加载脚本的所有要求。任何您已指定的参数为 必需 或 可选 将在脚本中显示要定义的接口。 在身份验证过程中，可以在脚本中使用它们的值，如为身份验证脚本提供的示例所示。 在对认证脚本所需的参数进行任何更改之后，请确保重新加载该脚本。 否则，接口中显示的参数可能不是在身份验证期间使用的，并且可能会发生错误。 The 凭证 used for each User during the authentication processed can be also specified in the Authentication Script and are configured in the 用户 标签。 读取更多...
