Opsyon ng aktibong iskan input ng mga vector screen

Ang screen na ito ay pinapayagan ka na i-configure ang aktibong iskan ng input ng mga vector.
Ito ang mga elemento na ang aktibong iskaner ay umaatake.
Ang pag-iskan ng lahat ng nagsusuportang mga elemento ay may katagalan, pero ang hindi pagiskan ng ibang elemento ay magiging dahilan ng mga kahinaan ay maaaring pumalya.

Naiinyek na mga target

Ang mga kahilingan ng mga elemento na ang aktibong iskaner ay ang target:

Ang URL ng Query String & Data Driven ng mga Node

Ang halaga ng mga paris na key sa aplikasyon na URL ng query, ang ie pagkatapos ?.
Kung walang query ng mga parameter kung magkagayon ang isa sa arbitaryo ay maaaring nasubukan na. kung Ang data ng mga driven node ay liwanagin sa loob ng isang context ito ay maaaring nasubukan na.

POST ng data

Ang halaga ng mga pares na key sa aplikasyon ng post data.

Daanan ng URL

Ang daanan ng mga elemento sa aplikasyon ng URL, ang ie ng mga elemento ay nakahiwalay sa pamamagitan ng /.

Mga header ng HTTP

Aplikasyon ng mga Header ng HTTP.
Lahat ng Aplikasyon
Pahintulutan para maiskan ang mga Header ng HTTP para sa lahat ng aplikasyon. Hindi lang mga aplikasyon na ipinadala sa mga parameter. kundi sa pamamagitan ng query o katawan ng aplikasyon.

Cookie ng Data

Aplikasyon ng mga cookie.

Build-in Input mga Handler ng Vector

Ang mga format ng data na target ng aktibong iskaner:
    Ang multipart na paraan ng data
    Ang XML tag/attribute
    JSON
    Ang Google Web Toolkit
    Ang OData id/filter

Paganahin ang iskrip input ng mga vector

Kung ang opsyon na ito ay napili na kung magkagayon ang aktibong iskaner ay maaaring gamitin ang kahit anong iskrip input ng mga vector.
Ang iskrip input ng mga vector ay mga iskrip na kung saan mayroon o importante sa ZAP at pinapahintulutan kang i-target ang mga elemento na kung saan ay hindi suportado ng default.

Ganoon din ang screen ay pinapahintulutan ka para i-configure ang mga parameter na kung saan ay maaaring baliwalain ng aktibong iskaner.
