Ang mga sesyon konteksto iskren

Ang mga iskren ang nagpapahintulot sa iyo na pamahalaan ang contexts.

Dito nakahanay ang mga iskren ng bawat konteksto iyong tutukoyin.

Ang Top iskren

Ito ay nag papahintulot sa iyo na lagyan ng pangalan ang konteksto at mailarawan.

Kasama sa konteksto

Ito ay nagpapahintulot sayo na pamahalaan ang URL na kung saan kasama ang konteksto.
Ang mga URL na hindi tumugma sa anu mang mga regex ay hindi kasama sa konteksto.

Ihiwalay mula sa konteksto

Ito ay nagpapahintulot sayo na pamahalaan ang mga URL na kung saan hindi kasama mula sa konteksto.
Ang kaylangan mo lamang upang matukoy ang mga regex o mga URL na ayaw mong isama pero tumugma ito sa isa o higit pang mga 'kasama' sa regexe.

Tignan ang Sesyon konteksto istractural iskren.

Teknolohiya

Ito ay nag papahintulot sa iyo na tukoyin ang mga teknolohiyang ginamit sa konteksto, Kung alam.
Ayon sa default ang lahat ng mga teknolohiya ay kasama.
Kung ihihiwalay ang mga teknolohiya na Alam mong Hindi ginagamit ma aaring bumilis ang up aktibong pag-skan sa mga patakaran tiyakin na maihihiwalay ang mga teknolohiya dahil puwedi itong malaktawan.

Ang mga gumagamit

Ito ay nagpapahintulot sayo na maisaayon ang set ng mga gumagamit para ito ay magamit pa sa iba't ibang gawain sa buong applikasyon.

Ang mga kredensiyal seksiyon ng mga gumagamit ay nakadipende sa Otentikasiyon method sa pag aayos ng konteksto.

Ang mga regex ng URL

Sa mga Naisama sa *, Mga nahiwalay mula * sa mga panel at ang Lagged in/out ay nagpapahiwatig at ang Otentikasyon panel, maaari mong ma pasok regular ang mga exprassiyon para matukoy ang hihiwalay ng mga URL. Habang ikaw ay umaalis sa nagiisang meta-karakter kasama ang backslash, maaari murin magamit ang \Q...\E umalis sa pagkakasunod sunod. Ang lahat ng mga karakter ay sagitna ng \Q at ng \E ang interpreted ay literal na karacter ng. E.g. \Q*\d+*\E magtutugma ang literal na teksto sa *\d+*. Ang pag alis ng pagkakasunod sunod ay magagamit sa ZAP Kung hindi ikaw kasama say mga URL sa pamamagitan ng parehong konteksto mga menu.
Tandaan: Kung ang iyong URL ay naglalaman ng "\E", at kaylangan mong sunden ang mga gagawin kung gagamit ka ng \Q...\E nasira ang pagkakasunod-sunod:
  • Buksan ang nasirang pagkakasunod-sunod
  • Isarado ang nasirang pagkakasunod-sunod bago ang "karakter" \E
  • Alisin ang backslash
  • Buksan pagkatapos ng "\E" ang iba pang naalis sa pagkakasunod-sunod;
  • Isarado ang nasirang pagkakasunod-sunod tulad ng karaniwan.

Halibawa: subdomain.example.com/path?a=\E&moredata=2 ay dapat ng lumitaw bilang \Qsubdomain.example.com/path?a=\E\\E\Q&moredata=2\E
