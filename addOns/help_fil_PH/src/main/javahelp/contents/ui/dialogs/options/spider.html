Mga opsyon sa iskrin ng Spider

Ang iskrin na ito ay pinahihintulutan ka na isaayos ang spider na mga opsyon.

Dapat itoy makilala na ang sakdalang pagsasaayos ng bahagya nitong mga opsyon ay ma-aapektohan ang gumaganang Spider.

Ang pinaka mataas na kalaliman sa pag gapang

Ang paramiter ay tumutukoy sa pinakamataas na kalaliman ng pag-gapang na kung saan ang pahina ay makikita sa kautusan para dito upang mai-proseso na. Nakita na ang mga Pinagmulan ang mas malalim pa sa level na ito ay kinuha at nai-parse na sa pamamagitan ng spider.

Ang lalim ay kinalkula na ito ay nag-simula mula sa mga binhi, kaya, kapag ang spider scan ay magsisimula ng nagiisa lamang na URL (eg. Spider URL), ang kalaliman ay nakalkula na mula sa isang ito. Gayon pa man, kapag ang scan ay nagsimula ng may maraming mga binhi (eg. Ang site ng spider), ang pinagkukunan ay naka proseso kung ang kalaliman nito ay nababagay sa anuman sa mga binhi ay mas mababa kesa sa isa pang nilarawan.

Bilang ng mga thread na nagamit

Ang spider ay maraming mga thread at ito ang bilang na nilalarawan ang napakaraming bilang ng gumaganang thread na ginagamit sa pag gapang na proseso. Baguhin itong paramiter na walang anumang epekto sa kahit anong pag-gapang na nasa pag unlad.

Pinakamataas na durasyon

Ang pinaka-mahaba na oras na ang spider ay gumana para sa, sinukat na mga minuto. sero (ang default) na nanganga-hulugang ang spider ay gumana hanggang sa makita ang lahat ng mga link na pwede sa.

Pinakamataas na anak para gumapang

Ang paramiter na ito ay nililimitahan ang bilang ng mga bata na gumagapang sa bawat node na nasa puno.
Ito ay kapaki pakinabang para sa mga aplikasyong pinapatakbo ng datos na mayroong malaking mga bilang ng 'mga pahina' ito ay katotohanan eksakto ang kaparehas na code ngunit naglalaman ng ibat ibang datos. halimbawa mula sa database.
Sa pamamagitan ng default ito ay maitatakda sa sero na nanganga-hulugang wala itong limitasyon na ginamit sa bilang ng mga buhol ng mga batang gumagapang.

Pinaka-mataas na sukat ng parse

Ipaliwanag ang pinakamataas na sukat, sa bytes, na ang tugon ay kailangan i-parsed. Pinapayagan nito ang spider upang laktawan ang malaking mga pagtugon/mga files.

Ang Domain ay laging nasa Abot ng isip

Payagan na pamahalaan ang string na literals ng domain o karaniwan na mga ekspresyon, na nasa abot ng kaisipan ng spider. Ang normal na ugali ng spider ay magfollow lamang ng mga link sa mga mapagkukunan na nakita sa kaparehas na saklaw tulad ng pahina kung saan ang scan ay magsisimula. Gayon man, ang opsyon na ito ay pinapayagan ka na ipaliwanag ang mga karagdagang mga domain na itinuturing "sa abot ng isip" habang nasa proseso ng pag-gapang. Mga pahina dito sa mga domain ay naka-proseso habang nagii-scan.

Mga Pag-usisa sa pag-kontrol ng mga paramiter

Kapag gumagapang na, ang Spider ay mayroong nasa loob na mekanismo na markahan kung alin ang mga pahina na tapos ng bisitahin, kaya hindi na ulit sila mag po-proseso. Nang itong pagsusuri ay nagawa, ang pamamaraan ng mga paramiter ng mga URI ay hinawakan ito ay itinakda gamit ang opsyon na ito. Mayroon ditong tatlong magagamit na mga opsyon:
  • Lubos na wag pansinin ang mga paramiter - kapag ang www.example.org/?bar=456 ay binisita, tapos ang www.example.org/?foo=123 ay hindi binisita
  • Ang kinikilala lamang ay ang pangalan ng paramiter (huwag pansinin ang halaga ng perimiter) - kapag ang www.example.org/?foo=123 ay binisita, tapos ang www.example.org/?foo=456 ay hindi binisita, ngunit ang www.example.org/?bar=789 o kaya www.example.org/?foo=456&bar=123 ay binisita
  • Kinikilala ang dalawa ang pangalan at halaga ng parameter - kapag ang www.example.org/?123 ay binisita, ang anumang ibang uri ito ay magkaiba (kasama ang, para sa halimbawa, www.example.org/?foo=456 or www.example.org/?bar=abc) ay binisita

Magpadala ng "Tumutukoy sa" pinuno

Kapag ang mga paki usap ay naipadala na ng may "Tumutukoy" sa pinuno.

Tanggapin ang mga Cookie

Kapag ang mga scan ng spider ay tinanggap ang mga cookie habang nag spidering. Kung maaari ang spider ay maayos na hahawakan ang mga natanggap na cookie mula sa server at ipapadala nila pabalik ng naaayon. Kung ang opinyon ay hindi pwede, ang Spider ay hindi magpapadala ng kahit anong mga cookie sa kahit anong mga paghiling. Para sa halimbawa, itoy kino-kontrol kung oo o hindi ang Spider ay gumagamit ng parehas na sesyon habang labas ang pag-spidering scan.
Kapag tumanggap ng mga cookie ang mga cookie ay hindi ibinahagi sa pagitan ng mga pag-scan sa spider, ang bawat pag-scan ay mayroong sariling banga ng cookie.
Ang opsyong ito ay maymababang prioridad, ang Spider ay irerespeto ng iba (pandaigdigan) mga opsyong kaugnay sa HTTP state. Ang opsyong ito ay hindi pinansin kung, para sa halimbawa, ang opsyon Paganahin ang Sesyon sa Pagsubaybay (Cookie) ay paganahin, kapag nag-spidering katulad ng Gumagamit o kapag ang HTTP Sesyon ay aktibo.

Mga form sa pag proseso

Habang nag po-proseso ang pag gapang, ang ugali ng spider kapag itoy makakatagpo ng HTML na mga form ay nililinawsa pamamagitan ng opsyon na ito. Kapag hindi pwede, ang mga anyo ng HTML ay hindi magpo-proseso sa lahat. Kung gumagana, ang mga form ng HTML na may paraan na isinasaad sa HTTP GET ay isinusumite na mayroong ilang lumilikha ng mga halaga. Ang ugali kapag nag-engkwentro ang mga form na may paraang nagpapalinaw tulad ng HTTP POST ay na-kumpigur sa mga susunod na opsyon.

Mga form sa POST

Tulad ng agarang pag lalarawan ng mga nakaraang talata (Mga form ng Pag-proseso), itong opsyon ay nagkukumpigur ng ugali ng spider kapag Mga form sa Pag=proseso ay gumagana at kapag nagkatagpo ang mga form ng HTML na mayroon para makapag POSTed.

Mga komento sa Parse HTML

Ang opsyon ay nagpapaliwanag kung ang spider ay spider ang HTML pagsusuri sa mga komento para sa mga link hanggang sa pinag-kukunan. Ang mapagkukunan lamang ang nakatagpo dito sa mga balidong komento HTML ang pag-tag ay nakaproseso.

Parse 'robots.txt' mga file

Ang opsyon na ito ay nagpapaliwanag kung ang spider ay spider ang robots.txt mga file na makikita sa mga website, magsiyasat tungkol sa mga link sa mga pinagkukunan. Ang opsyong ito ay hindi ipinapaliwanag kung ang spider ay sumusnod sa mga kautusan inilagay ng robots.txt file.

Hawakan ang OData-natatangi mga paramiter

Ang opsyong ito ay nagpapaliwanag kung ang spider ay susubukan na alamin ang natatanging mga paramiter ng OData (i.e. pinagkukunan tagapagpaliwanag) sa utos na maayos na i-proseso nila ayon sa mga rule na kanilang ipinaliwanag sa pamamagitan ng "Humahawak sa mga paramiter ng pag-usisa" opsyon.
