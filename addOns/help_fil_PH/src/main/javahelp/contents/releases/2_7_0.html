Inilabas na 2.7.0

Ito ay pagsasaayos ng isang bug at mas papagandahin ang paglabas nito, kung saan ang Java 8 ay nangangailangan ng minimum.

Mga ilang makabuluhang mga pagsasaayos kasama ang:
  • Kasama ng default ang inilunsad na Browser - pinapayagan ka nito na ilunsad ang mga browser mula sa ZAP para sa pagsasaayos ng kinatawan sa pamamagitan ng ZAP at pabayaan ang sertipiko ng mga babala dahil sa ugat ng sertipiko ng ZAP.
  • Payagan ang ZAP na makinig ng maraming mga adres/ports
  • Suportahan ang indikasyon ng pangalan ng serber
  • Nabagong NTLM engine implementation - ang mga kasong mga isinasaayos nito kung saan ang dominyo ay napatunayan at pinagbuti ang interoperability kasama ang ibang mga (serber) NTLM implementations
  • Maraming mga bagong API endpoints - tingnan ang mga detalye sa ibaba
Tandaan kung meron kang anumang mga problema dito sa inilabas na ito kung gayon mayroon ring bagong 'Impomasyong Tulong/Suporta...' Ang pagpipilian ng mga gamit upang ang mga ilalaan ay ang mahahalagang impormasyon tungkol sa iyong instilasyon ng ZAP kung saan kailangan mong magsama ng kahit anung mga isyu na nilikha.

Ang labag na mga pagbabago ng ZAP API:

Awtentikasyon ng AKSYON / setAuthenticationMethod

Ang tipo ng awtentikasyon "formBasedAuthentication" ay kinakailangan ngayon ang pag-login sa URL sa bumubuo ng encoded lage. Ang pagbabago ay tinitiyak na ang login sa URL ay ginamit/pinadala bilang ito ay tinukoy. Noon tinatanggap nito ng bahagyang na-encode na mga URL pero muli nilang ini-encode kapag ginamit na, potensyal itong patungo sa iba't ibang URL na ginamit/pinadala.

Tingnan ang Konteksto / contextList

Itong pagbabago ay makakasira ng mga mamimili na inaalisa ang salita ng manwal/kuhanin ang mga pangalan mula sa string. Ang straktura ng datos ay ibinalik ng maayos na pagbabago na nakahiwalay ang pangalan: 

{"contextList":["konteksto 1","Konteksto 2"]}
at: 

<contextList type="list"> <contextName>Kontekstong isa</contextName> <contextName>Kontekstong dalawa</contextName> </contextList>
sa halip na: 

{"contextList":"[Kontekstong isa, Kontekstong dalawa]"}
at: 

<contextList>[Kontekstong isa, Kontekstong dalawa]</contextList>

Tingnan ang buod / setOptionUseProxyChain

Binago para bumalik BIGO kung ang kinatawan ay outgoing ito ay hindi pinapagana (Dahil ang kailangan na adres/hostname ay hindi nai-set noon), bago bumalik palage ito OK.

ZAP API binago ang mga Endpoint:

Tingnan ang buod / mga alerto

Idinagdag na pagpipilian ang riskld parametro sa pagpapadali ng pagsala ay mapanganib. Kung saan ang riskld ay nasa 0 na nagbibigay ng kaalaman at 3 para sa mataas.

Tingnan ang buod /BilangNgMgaAlerto

Idinagdag na pagpipilian ang riskld parametro sa pagpapadali ng pagsala ay mapanganib. Kung saan ang riskld ay nasa 0 na nagbibigay ng kaalaman at 3 para sa mataas.

Tingnan ang buod / mga url

Magdagdag ng pagpipilian baseurl parametro, sa pagsala ng mga URL ay bumalik.

Tingnan ang ascan / mga pagsuri

Binago na rin ang pagbalik ng kabuuang bilang ng mga alerto na itinaas at mga mensahe na pinadala sa panahon ng bawat pagsuri.

Tingnan ang ascan / scanProgress

Binago na rin ang pagbalik ng kabuuang bilang ng mga alerto na itinaas sa pamamagitan ng bawat nagsusuri.

ZAP API Ang bagong mga Endpoint:

Tingnan ang buod / alertsSummary

Ang isang bagong makikitang lagom para sa mga alerto ay ang mga dini-display ng mga bilang ng mga alerto bawat antas ng panganib. Opsyonal, naisala sa pamamagitan ng halaga ng baseurl. 

{"Mataas":0,"Mababa":132,"Katamtaman":39,"Impomasyon":153}

Tingnan ang buod / messagesByld

Ang mga pagkuha ng mga mensahe ng HTTP kasama ng ibinigay na mga ID.

Tingnan ang buod / messagesHarByld

Ang mga pagkuha ng mga mensahe ng HTTP kasama ng ibinigay na mga ID, sa pag-ayos ng HAR.

Tingnan ang buod /optionAlertOverridesFilePath

Ang mga pagkuha ng mga daan sa payl kasama ng mga override na alerto.

Tingnan ang buod / optionMaximumAlertInstances

Ang mga pagkuha ng pinamataas na bilang ng alerto ng mga instance para isama sa isang ulat.

Tingnan ang buod / optionMergeRelatedAlerts

Kung ang mga pagkuha o hindi na kaugnay ang mga alerto na maaaring ipagsama sa kahit na anung mga ulat na binubuo.

Tingnan ang buod / zapHomePath

Ang mga pagkuha ng daan sa tahanan ng ZAP directory.

Tingnan ang mga localProxy / mga additionalProxy

Ang mga pagkuha ng lahat ng mga karagdagang mga kinatawan na na-configure.

Tingnan ang ispayder / optionAcceptCookies

Kung ang mga pagkuha o hindi prinoseso ang ispayder ay kailangan na tanggapin ang mga cookie habang nag-i-ispaydering.

Ang Aksyon ng buod / deleteAlert

Ang alerto ay mga binura kasama ang ibinigay na ID.

Ang Aksyon ng buod / setOptionAlertOverridesFilePath

Ang mga itatakda (o mga klaro, kung wala ng laman) ang daan sa payl kasama ang mga override.

Ang Aksyon ng buod / setOptionMaximumAlertInstances

Ang mga itatakdang pinakamataas ang bilang ay alerto ng mga instance sa pagsama sa isang ulat. Ang halaga ng sero ay itinuturing na walang limitasyon.

Ang Aksyon ng buod / setOptionMergeRelatedAlerts

Kung ang mga itatakda o hindi kaugnay ng mga alerto na maaaring masama sa kahit anung mga ulat na binubuo.

Ang aksyon ng ascan / importScanPolicy

Ang patakaran ng pagsuri ay mga angkat gamit ang ibingay na payl sa daan ng sistema.

Ang Aksyon ng ascan / skipScanner

Ang nagsusuri ay may mga laktaw gamit ang mga ID ng pagsuri at ng nagsusuri.

Tingnan ang mga localProxy / addAdditionalProxy

Ang isang bagong kinatawan na mga idinagdag ang mga detalye na isinuplay. Tingnan ang Mga pagpipilian ng lokal na mga kinatawan ng iskrin para sa mga detalye ng mga parametro.

Tingnan ang mga localProxy / removedAdditionalProxy

Tanggalin ang mga nadagdag na kinatawan kasama ang tinukoy na adres at port.

Ang aksyon ng ispayder /mga setOptionAcceptCookie

Kung ang mga itinakda o ang hindi pagproseso ng ispayder ay hindi dapat tanggapin ang mga cookie habang nag-i-ispaydering.

Ang mga detalye ng mga kahinaan

Ang mga sumusunod na mga kahinaan ay inuulat sa dating bersyon ng ZAP.
Maraming salamat sa lahat ng mga mananaliksik na may mga etikal na ulat na mga isyu para sa atin sa pamamagitan ng ating bug na programang mapagbigay.
Kung kailangan ninyo ng mga detalye tungkol dito sa kahinaan mangyari lamang na makipag-ugnayan po kayo sa amin.

Ang mga window na mga taga alis ng mga isinalin na mga kahinanaan sa DLL Hijacking

Ang mga ZAP window na taga alis ng mga isinalin para sa 2.6.0 ay mahina sa DLL Hijacking sa mga window. Ito ay kahinaan sa pangatlong partido na taga salin ng install4j na ngayon ay naisaayos na.
Tandaan na ito ay magaganap lamang kung ang malisosyong DLL ay nasa daan na mismo.
