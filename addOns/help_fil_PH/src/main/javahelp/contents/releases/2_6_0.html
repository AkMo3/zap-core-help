Inilabas 2.6.0

Ang inilabas na ito ay naglalaman ng isang malaking bilang ng mga pagbabago, partikular na ang ZAP API.
Kami ay nagdagdag ng isang makabuluhang bilang ng mga bagong endpoints ng API, nagtatrabaho patungo sa aming layunin ng paggawa ng ZAP ganap na pagpigil sa pamamagitan ng API. Binago rin namin ang ilan sa mga umiiral na endpoints at sa lahat ng mga kaso ang mga pagbabagong ito ay pabalik na magkatugma.

Ang buong pag pakawala ay kabilang din ang bagong JxBrowser add-on pati na rin ang platform specific webdricvers para ito ay magawa ng mas madali.

Mga pagbabago sa seguridad ng API

Binago namin ang seguridad ng API bilang tugon sa mga isyu na iniulat sa atin sa pamamagitan ng bug bounty program. Nakatakda sa ibaba ang mga detalye ng mga kahinaan.
Ang mga pagbabago ng seguridad ay sa pangangailangan na hindi paurong na tugma, bagama't isinama namin ang mga opsiyon para sa hindi pagpapagana sa kanila kung gagamit ka ng ZAP sa isang ligtas na paligid.

Sa pamamagitan ng pag-default sa lahat ng tawag ng API ngayon ay kailangan na nang API key o ng nonce. Ito ay maaaring magbigay sa pamamagitan ng URL parameter, POST parameter o mga header. Ang supportadong kliyente ng ZAP API (kabilang ang Java at Python) ay naging bago sa nagbibigay ng API key sa pamamagitan ng header. Ang nonces ay nabuo sa pamamagitan ng ZAP at nilalayon na upang gamitin sa pamamagitan ng ZAP add-ons na kailangan sa pag-akses ng ZAP API. Para sa buong detalye tingnan ang Tabing ng API ng mga pagpipilian.

May mga bagong hanay na pagpipilian sa API na may kaugnay sa seguridad:

  • UI Enabled - Kung pinagana pagkatapos ang API Web UI ay magagamit sa lahat ng mga makina na magamit ang ZAP bilang proxy. Ito ay pinagana sa pamamagitan ng pag-default.
  • Ang mga IP address ay pinahihintulutan upang gamitin ang API - Sa pamamagitan lamang ng pag-default ng makina na tumatakbo sa ZAP ay magagawang mapasok ang ZAP API. Maaari mong ipahintulot ang ibang makina na pumasok sa API sa pamamagitan ng pagdagdag ng mga angkop na disenyo ng regex. Kailang mo lang magdagdag ng mga IP address na iyong pinagkakatiwalaan.
  • Hindi nangangailangan ng isang API key para sa mga ligtas na operasyon - Kung pinagana pagkatapos ang API key ay hindi kinakailangan para sa pagtingin o iba pang operasyon na isinasaalang-alang ang 'kaligtasan', sa madaling salita ang mga operasyon na hindi gumawa ng anumang pagbabago sa ZAP. Gayun paman ang mga naturang operasyon ay nagbibigay nang-access sa ZAP data tulad ng mga alerto, mga mensahe at mga landas ng sistema ng file. Maaari din itong gamitin sa pamamagitan ng mga web application upang makita ang presensya ng ZAP.
  • Iulat ang mga error permiso sa pamamagitan ng API - kung panagana pagkatapos ang ZAP ay mag-uulat ng mga error sa permiso sa pamamagitan ng API, na maaaring gamitin ng mga web application para tuklasin ang kinaroroonan ng ZAP. Ito ay hindi isang seryosong problema sa isang ligtas ng kapaligiran pero kung ikaw ay gumagamit ng ZAP labang sa posibleng malisiyosong mga site kung ganoon hindi mo dapat paganahin ito.
Lahat ng mga ZAP opsyon ay maaaring tukuyin sa pamamagitan ng command line pagka sinimulan mo na ang ZAP - tignan ang API Key FAQ para sa buong mga detalye.
Nagdagdag din kami ng marami pang mga security header sa API kasama ang isang matibay ng Content Security Policy.

Pagbabago sa mga Endpoint ng ZAP API:

Aksyon ng ascan / scan

Ang url parameter ay opsyonal na ngayon at ang isang opsyonal na contexttld parameter ay idagdag na. Kinakailangan mong suportahan ang isa sa mga yon.

Aksyong ng ascan / scanAsUser

Ang url at contextld parameter ay opsyonal na ngayon. Kailangan mong suportahan ang isa sa mga yon.

Aksyon ng ascan / addScanPolicy

Idinagdag na mga opsyon ng alertThreshold at attackStrength parameter.

Bagong mga Endpoint ng ZAP API:

Ipakita ang ascan / optionMaxRuleDurationInMins

Ibalik ang pinakamataas na oras sa minuto kung saan ang isang karaniwang scan ay maaaring mapagana sapagkat, ang zero ay walang limitasyon.

Ipakita ang ascan / optionMaxScanDurationMins

Ibalik ang pinakamataas ng oras sa minuto kung saan ang buong scan ay maaaring mapagana sapagkat, ang zero ay walang limitasyon.

AKSYON ascan / setOptionMaxRuleDurationInMins

I-set ang pinakamataas na oras ng minuto na itatagal ng paggana ng scan rule, ang zero ay walang limitasyon.

AKSYON ascan / setOptionMaxScanDurationInMins

I-set ang pinakamataas na oras ng minuto na itatagal ng paggana ng scan rule, ang zero ay walang limitasyon.

AKSYON ascan / updateScanPolicy

I-update ang tinukoy na polisiya ng scan na may tinukoy na alertThreshold o attackStrenght.

Ipakita ang bago / isBreakALL

Ibalik sa TAMA kung babaguhin ng ZAP ang parehong mga kahilingan at mga tugon.

Ipakita ang bago / isBreakRequest

Ibalik sa TAMA kung ang mga kahilingan ay babaguhin ng ZAP.

Ipakita ang bago / isBreakResponse

Ibalik sa TAMA kung ang mga tugon ay babaguhin ng ZAP.

Ipakita ang binago / httpMessage

Ibinabalik ang mensahe ng HTTP na kasalukuyang nagambala (kung mayroon man).

AKSYON break / break

Kinokontrol ang pag-andar ng mga global break. Ang uri ay maaaring isa sa: http-all, http-request o http-response. Ang estado ay maaaring totoo (para sa pagpihit ng break sa on para sa tinutukoy ng uri) o huwad (para sa pagpihit ng break sa off). Sa kasalukuyan ang Scope ay hindi ginagamit.

AKSYON break / setHttpMessage

Pinapalitan ang kasalukuyang mensahe na naharang kasama ang ibinigay na datos.

AKSYON break / continue

Isumite ang kasalukuyang naharang na mga mensahe at i-unset ang global request/response sa mga break point.

AKSYON break / step

Isumite ang kasalukuyang naharang na mensahe, ang susunod na kahilingan o tugon ay automatikong haharangin.

AKSYON break / drop

Inilalagay ang kasalukuyang mensahe na naharang.

TIGNAN ang core / optionDnsTtlSuccessfulQueries

Makakakuha ng TTL (sa ilang mga segundo) ng matagumpay ng mga query ng DNS.

AKSYON core / sendRequest

Nagpadala ang HTTP ng Kahilingan, opsyonal na pagsunod sa mga redirection. Ibalik ang kahilingan naipadala at tugon na natangap at sumunod sa mga redirection, kung meron man. Ang Mode ay ipinatutupad kapag nagpapadala ng kahilingan ( at kasunod ang redirection), ang mga pasadyang manu-manong kahilingan ay hindi pinapayagan sa 'ligtas' na mode kahit sa 'protektadong' mode kung labas sa scope.

AKSYON core / setOptionDnsTtlSuccessfulQueries

I-set ang TTl (sa mga segundo) ng matagumpay na mga tanong ng DNS (gamitin gamitin pagkatapos na ang ZAP ay mai restart).

Iba pang core / mdreport

Bumubuo ng isang ulat sa Markdown format.

TIGNAN ang mga httpSession / mga site

Kunin ang lahat ng mga site na mayroong mga session.

TIGNAN ang pscan / scanOnlyInScope

Sabihin kung ang passive scan ay dapat bang gumanap o hindi sa mga mensaheng na nasa scope.

AKSYON pscan / setScanOnlyInScope

I-set kung ang passive scan ay dapat bang gumanap lang sa mga mensaheng na nasa scope o hindi.

TIGNAN ang ispayder / allurls

Ibalik ang isang listahan ng mga natatanging URL mula sa talahanayan ng kasaysayan batay sa mga mensaheng idinagdag ng ispayder sa HTTP.

TIGNAN ang Ispayder / optionMaxChildren

Kunin ang pinakamataas na bilang ng mga child node (bawat node) na maaaring i-crawled, 0 ibig sabihin walang limitasyon.

AKSYON ispayder / setOptionMaxChildren

I-set ang pinakamataas na bilang ng mga child node (bawat node) na maaaring i-crawled, 0 ibig sabihin ay walang limitasyon.

Mga Detalye ng kahinaan

Ang mga sumusunod na kahinaan ay naiulat na sa nakaraan mga bersyon ng ZAP. Ang iba pang hindi gaanong seryosong isyu ay na ayos na bilang isang bagay ng course.
Maraming salamat sa lahat ng mga mananaliksik na mayroong etkal na ulat ng mga isyung ito sa amin sapamamagitan ng ating bug bounty program. kung kailangan mo ng karagdagang mga detalye tungkol sa kahit aling kahinaan mangyari na makipag-ugnayan po kayo sa amin.

RCE sa pamamagitan ng Anti CSRF Test Form at API Key Disclosure

Kung ang gumagamit ay gumamit ng Anti CSRF Test Form laban sa isang partikular na ginawang HTML page at pagkatapos ang API key ay nahayag sa site na iyon. Pagkatapos na ma-access ang site ang ZAP API ay magsasagawa ng anumang aksyon, kabilang ang pag-upload ng mga ZAP script. Ang mga Iskrip ay maaaring lamang i-upload sa mga 'lokal' na filesystem subalit kung ang gumagamit ay nagpapaandar ng ZAP sa Windows kung gayon ang attacker ay maaaring makagawa ng isang malisyosong iskrip na magagamit sa pamamagitan ng isang pampublikong SMB share. Ito ay lilitaw sa ZAP bilang isang lokal na payl at ang iskrip ay na-upload at maaaring mapagana sa pamamagitan ng API.
Ang kinakailangan para sa API key o nonce sa lahat ng mga operasyon ng API ay isang direktang resulta ng kahinaan na ito, gaya ng pagbabago ng mga add-on sa paggamit ng nonce upang mabawasan ang panganib ng pagtagas ng API key.

Kredito: Artemy Bogdanov (@Abr1k0s)
Si Artemy ay ginawaran ng isang award ng $1000 bug bounty bilang resulta ng kanyang pagsusumite. ito ang unang bug bounty na aming binayaran - congratulation Artemy!

Windows Installer Vulnerable sa DLL Hijacking

Ang ZAP Windows Installer para sa lahat ng mga bersyon hanggang sa kabilang ang 2.5.0 ay mahina sa DLL Hijacking sa Windows 7 (at mga unang bersyon). Ito ay isang kahinaan sa 3rd party installer InnoSetup. Ang 2.6.0 na mga installer (sa lahat ng platforms) ay nabuo na ngayon gamit ang Install4J.

Kung sa ilang kadahilanan na kailangan mong i-install ang lumang bersyon ng ZAP sa Windows 7 o mas maaga kung ganon ay nirerekuminda namin sa iyo na ilipat mo ang installer sa isang malinis na direktaryo bago paandarin ito.

Tandaan na ang Burp Suite ay gumagamit din ng Install4J sa hinaharap na kahinaan sa Install4J-generated na mga installer maaaring karapat dapat para sa Burp Suite bug bounty program: https://hackerone.com/portswigger

Kredito: James Kettle (Burp Suite)

Kodigo ng di-makatwirang pagpapatupad sa pamamagitan ng mga invoke Application Parameter Injection

Ang mga parameter ng HTML ay maaaring maging partikular na ginawa upang maging sanhi ng di-makatwirang pagpapatupad ng code, kung pipiliin ng gumagamit na tumawag sa naka-target na application na may isang kahilingan na naglalaman ng parameter mula sa ZAP.
Ang mga Invoke Application Add-on ay na-update na para maayos ang isyu - lahat ng mga gumagamit ng ZAP ay kailangan install itong bagong bersyon bago magpatuloy sa paggamit ng add-on.

Kredito: Artemy Bogdanov (@Abr1k0s)

XSS sa pamamagitan ng CSRF Test Form

Ang Anti CSRF Test Form ay walang tibay sa mga atake ng XSS kung tumatakbo laban sa tinutukoy na ginawang HTML page.
Gumagamit ngayon ang API ng isang malakas na patakaran ukol sa seguridad ng naglalaman para maiwasan ang gayong mga isyu

Kredito: g_sato - https://bugcrowd.com/g_sato

API na mahina sa DNS Rebinding

Ang API ay mahina sa mga atake ng DNS Rebinding. sinusuri nito ngayon ang host header at tinatanggihan ang anumang kahilingan mula sa mga hindi inaasahang host.

Kredito: Artemy Bogdanov (@Abr1k0s)
