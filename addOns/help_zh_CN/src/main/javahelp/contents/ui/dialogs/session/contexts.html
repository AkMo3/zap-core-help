会话上下文屏幕

这些屏幕允许您管理上下文.

您定义的每一个上下文都有一组屏幕。

顶部屏幕

这允许您设置上下文名称和描述。

包含在上下文中

这允许您管理将包含在上下文中的URL。
不匹配任何正则表达式的URL将不包含在上下文中。

从上下文中排除

这允许您将管理从上下文中排除的URL。
You only need to specify regexs for URLs that you do not want to include but which match one or more of the 'include' regexes.

结构

参阅 会话上下文结构屏幕.

技术

这允许您指定上下文中使用的技术（如果已知）。
默认情况下，所有技术都包含在内。
If you exclude technologies that you know are not used then this may speed up 自动扫描 as rules specific to the excluded technologies can be skipped.

身份验证

参阅 会话上下文身份验证屏幕.

This allows you to manage the way in which Session Management is being done for the Context. 在选择了会话管理方法类型之后，需要配置的选项 取决于会话管理方法。

用户

这允许您配置一组用户，它们可以用于整个应用程序的各种其他操作。

用户的凭据部分取决于上下文中配置的身份验证方法。

URL正则表达式

在 包括在*, 从*中排除 面板和 已登录/注销指示器 of the 身份验证 面板，您可以输入正则表达式来定义排除的URL。虽然您可以反斜线转义单个元字符，您也可以使用 \Q...\E 转义序列。中所有字符 \Q 和 \E 被解释为原义字符。例如：\Q*\d+*\E匹配文本*\d+*。当您通过一些上下文菜单排除URL时，此转义序列用于ZAP。
备注： 如果您的URL包含一个＂\E"，那么当使用时您必须按照以下步骤做 \Q...\E 转义序列：
  • 打开转义序列
  • 在"字符"\E之前关闭转义序列
  • 转义反斜线
  • 在"\E"之后打开另一个转义序列；
  • 按正常方式关闭转义序列。

Example: subdomain.example.com/path?a=\E&moredata=2 should appear as \Qsubdomain.example.com/path?a=\E\\E\Q&moredata=2\E
