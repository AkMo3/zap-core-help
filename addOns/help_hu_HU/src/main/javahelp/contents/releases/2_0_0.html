Release 2.0.0

A következő változások történtek ebben a kiadásban:

Jelentős változások:

Integrált bővítmények piactér

ZAP meghosszabbítható bővítmények, amelyek teljes hozzáférés a ZAP belső. Bárki írhat bővítmények, illetve feltölteni őket a ZAP Add-on Piac (OK, akkor a Google code project nevű zap-fájlokat, de az ötlet).
Ami fontosabb, hogy most böngészhet, töltse le, majd telepítse azokat a bővítmények belül ZAP. A legtöbb bővítmény lehet dinamikusan telepített (eltávolított), így nem is kell indítani.
Beállíthatja, hogy értesítést a frissítést, akkor is automatikusan frissül. S mivel a vizsgálat szabályokat már végre, mint a kiegészítőket lehet kapni a legújabb szabályok, amint teszik közzé.

Utódja a "standard" Pók

A "régi" Pók volt, mutatja, hogy a kor, a teljesen újraírt, sokkal gyorsabb, átfogóbb, mint a régi. Ez még mindig egy "hagyományos" pók, amely elemzi a HTML kódot olyan linkeket találtok.

Egy új 'Ajax pók,

Amellett, hogy a "hagyományos" pók már hozzá egy Ajax pók, amely hatékonyabb alkalmazásokat, hogy a nehéz JavaScript használatát. This uses the Crawljax project which drives a browser (using Selenium) and so can discover any links an application generates, even ones generated client side.

Web Socket support

ZAP most már támogatja a WebSockets, így ZAP meg WebSocket küldött üzenetek, illetve a böngésző. Mint a HTTP-alapú üzenetek, ZAP is eltéríthetik a WebSocket üzeneteket, valamint lehetővé teszi, hogy megváltoztassa őket menet közben.
Azt is fuzz WebSockets üzenetek is használja a fuzzing rakományának tartalmazza ZAP a projektek, mint JBroFuzz, fuzzdb. Természetesen könnyen hozzá a saját fuzzing fájlokat.

Quick Start tab

Az első fő fülre, megjelenik a Quick Start' lap, amely lehetővé teszi, hogy csak írja be az URL-t, majd megvizsgálni, egy kattintással.
Ez egy ideális kiindulópont, hogy az emberek az új alkalmazás biztonsági, de a szakértők szerint könnyen eltávolítani, ha megtalálják zavaró.

Munkamenet-tudatosság

ZAP most ülésen figyelmet, hogy ZAP lehet felismerni nyomon követni a több menetben. Ez lehetővé teszi, hogy hozzon létre új ülés, váltani őket, vonatkozik mind az egyéb összetevők, mint a Pók, Aktív Szkenner.

Felhasználó által meghatározott Kontextusokban

Akkor most adjuk meg tetszőleges számú 'kontextusokban' kapcsolódó beállítja az Url-címek, amelyek egy alkalmazás. Akkor a cél Url egy olyan környezetben, mint például a pókoktól, vagy Aktív Szkenner. Azt is hozzá a kontextusban, hogy a hatálya alá, valamint a társult egyéb adatokat, például hitelesítési adatait.

Session scope

Az ülés hatálya lehetővé teszi, hogy meghatározza, hogy milyen kontextusban érdekel egy időben. A különböző fülek jelenít meg olyan tartalmat, csak az érvényes URL korlátozhatja, illetve a Védett üzemmód megakadályozza, hogy véletlenül támadó URL-eket.

Különböző módok

ZAP most már támogatja a 3 módok:
  • Biztonságos, amelyben nem a potenciálisan veszélyes művelet megengedett
  • Védett, amelyben elvégezheti azokat az intézkedéseket, az Url-ek hatálya
  • Szabvány, amely bármire képes, hogy bármilyen Url-ek

Authentication handling

Most már munkatárs hitelesítési adatait semmilyen összefüggésben, amely lehetővé teszi, ZAP, hogy a dolgok, mint felismerni, ha ön nincs bejelentkezve pedig automatikusan jelentkezzen be újra. Ez különösen akkor hasznos, amikor a használt keresztül az API biztonsági regressziós vizsgálatok.

More API support

A rest API jelentősen bővült, így sokkal hozzáférés a funkciót ZAP biztosít.

Finom szemcséjű szkennelés ellenőrzések

Az aktív szkennelési szabályok most már hangolt állítsa be az erejét (a támadások száma végeznek), valamint a küszöböt, amely a jelentés esetleges problémákat.

Új, továbbfejlesztett, az aktív, mind a passzív szkennelés szabályok
