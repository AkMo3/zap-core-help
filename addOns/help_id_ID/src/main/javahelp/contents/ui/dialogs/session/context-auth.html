Konteks sesi Otentikasi layar

Ini adalah salah satu dari Sesi Konteks layar yang memungkinkan kamu mengelola cara di mana Otentikasi sedang dilakukan untuk konteks. Perhatikan itu mengubah metode otentikasi setelah Pengguna telah ditetapkan akan menyebabkan mereka untuk dihapus, seperti jenis kredensial pengguna harus sesuai dengan skema otentikasi. Setelah memilih tipe metode otentikasi, pilihan yang perlu dikonfigurasi tergantung pada metode otentikasi.

Otentikasi Manual

Tidak diperlukan konfigurasi untuk metode otentikasi ini. Read more...

Otentikasi berbasis bentuk

Untuk mengkonfigurasi metode otentikasi ini, kamu harus menyediakan login urluntuk yang login permintaan ini dilakukan, permintaan tubuh (Mengirim data), jika diperlukan, dan mengidentifikasi parameter digunakan untuk memasok 'username' dan 'password'. Jika tidak ada permintaan tubuh disertakan, login permintaan ini dilakukan sebagai HTTP GET, jika POSTING HTTP yang digunakan. Kepercayaan diri dikonfigurasi dalam Users tab. Read more...

Otentikasi HTTP / NTLM

Untuk mengkonfigurasi metode otentikasi ini, anda perlu menyediakan nama tuan rumah dan port server otentikasi dilakukan dengan dan realm kredensial yang berlaku. Kepercayaan diri dikonfigurasi dalam Pengguna tab. Read more...

Skrip berbasis otentikasi

Untuk menggunakan metode ini, anda pertama perlu untuk menulis (dan menyimpan) Layanan otentikasi menggunakan Skrip tab (lihat diberikan contoh-contoh dan template untuk jenis script di Script tab). Kemudian anda perlu untuk memasok nama script dalam daftar dropdown. Setelah memilih naskah, anda perlu menekan yang Beban tombol, memuat semua persyaratan script. Setiap parameter yang telah ditetapkan sebagai required or optional dalam naskah akan akan ditampilkan dalam antarmuka untuk didefinisikan. Nilai mereka tersedia untuk digunakan dalam naskah, selama otentikasi, seperti yang terlihat dalam contoh yang diberikan untuk skrip otentikasi. Pastikan bahwa setelah melakukan perubahan pada parameter yang diperlukan oleh Otentikasi script anda re-load script. Jika tidak, parameter yang ditunjukkan di antarmuka mungkin tidak menjadi yang digunakan selama otentikasi dan kesalahan mungkin terjadi. The credentials digunakan untuk setiap Pengguna selama otentikasi diolah dapat juga ditentukan dalam Otentikasi Script dan dikonfigurasi di Users tab. Read more...
