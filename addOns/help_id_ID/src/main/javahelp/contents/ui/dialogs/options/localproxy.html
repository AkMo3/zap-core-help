Layar Pilihan Proxy Lokal

Pilihan Koneksi layar memungkinkan anda untuk mengkonfigurasi alamat dan port di mana ZAP menerima koneksi masuk.

Proxy lokal

Secara default ZAP akan mendengarkan pada satu alamat lokal dan port, dan ini adalah alamat dan port yang harus anda gunakan untuk mengkonfigurasi browser untuk menggunakan proxy.

Alamat

Alamat lokal yang akan digunakan ZAP. Semua alamat yang tersedia diidentifikasi secara otomatis.

Pelabuhan

Pelabuhan yang ZAP akan mendengarkan.

Dibelakang NAT

Menunjukkan bahwa Lokal Proxy (ZAP) adalah di belakang NAT. Ketika dipilih ZAP akan mencoba untuk menentukan alamat IP publik, untuk benar-benar mendeteksi dan menangani permintaan dengan publik Alamat IP (misalnya, untuk dilayani oleh ZAP API).

Catatan: Pilihan ini hanya didukung ketika ZAP berjalan di sebuah Contoh AWS EC2. ZAP akan mendapatkan alamat IP publik dari AWS EC2 instance's metadata.
ZAP harus dimulai dengan opsi ini diaktifkan jika akses ke API, melalui alamat IP publik, diperlukan:

zap.sh -daemon -port 8080 -host 0.0.0.0 -config proxy.behindnat=true
Also, the API perlu dikonfigurasi untuk menerima alamat IP eksternal (yaitu IP alamat dari mana ZAP sedang diakses).

Hapus Pengkodean yang Tidak ada dukungan

Memungkinkan proxy untuk menghapus yang tidak didukung pengkodean dari "Accept-Encoding" bidang header permintaan, sehingga tidak ada (tidak didukung) pengkodean dan transformasi yang dilakukan untuk respon.
Opsi ini harus selalu diaktifkan kecuali ketika pengujian transformasi encoding.
Pesan dikodekan dengan tidak didukung penyandiaksaraan akan tidak benar dipindai (baik dengan scanner pasif dan aktif).

Protokol keamanan

Memungkinkan untuk memilih SSL/TLS versi diaktifkan untuk masuk ke koneksi (misalnya, dari browser). Setidaknya salah satu versi harus diaktifkan, versi tidak didukung oleh JRE akan tidak dipilih dan dinonaktifkan.
Pilihan SSLv2Hello harus dipilih dalam hubungannya dengan setidaknya satu SSL/TLS versi.
Opsi keamanan berlaku untuk kedua utama dan semua proxy tambahan.

Tambahan proxy

Anda dapat menambahkan sebanyak alamat tambahan dan pelabuhan untuk ZAP untuk mendengarkan pada yang Anda inginkan.
Hal ini dapat berguna ketika pengujian aplikasi mobile - anda dapat proxy mobile aplikasi melalui antarmuka Wi-Fi pada saat yang sama dengan menggunakan backend situs ini diatas menggunakan browser proxy melalui ZAP meniru pengguna mobile agent.
