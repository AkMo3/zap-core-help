Pilihan layar masukan Vektor aktif

Layar ini memungkinkan anda untuk mengkonfigurasi aktif scan masukan vektor.
Ini adalah elemen pemindai aktif yang akan menyerang.
Memindai semua elemen yang didukung akan menghabiskan waktu yang lebih lama, namun tidak dapat memindai beberapa elemen bisa menyebabkan beberapa kelemahan yang dilewatkan.

Sasaran yang disuntikkan

Elemen permintaan yang ditargetkan untuk pemindai akan aktif:

String Permintaan URL & Data Didorong Node

Pasangan nilai kunci dalam kueri URL permintaan, yaitu setelah ?.
Jika tidak ada parameter kueri maka yang sewaktu-waktu akan diuji. Jika Data Didorong Node didefinisikan dalam konteks mereka akan diuji.

Data POST

Pasangan nilai kunci dalam permintaan data POST.

Jalur url

Elemen jalur dalam URL permintaan, yaitu elemen yang dipisahkan oleh /.

Header HTTP

Permintaan HTTP Header.
Semua permintaan
Memungkinkan untuk memindai Header HTTP dari semua permintaan. Tidak hanya permintaan yang mengirimkan parameter, melalui query atau permintaan body.

Data cookie

Permintaan cookie.

Membangun masukan menangani Vektor

Format data yang dipindai aktif akan menargetkan:
    Data Formulir Multipart
    Tag/atribut XML
    JSON
    Google Web Toolkit
    OData id/menyaring

Mengaktifkan Skrip masukan vektor

Jika pilihan ini dipilih maka pemindai aktif akan menggunakan vektor masukan skrip yang diaktifkan.
Vektor masukan skrip adalah skrip yang telah anda tulis atau impor ke ZAP dan memungkinkan anda untuk menargetkan elemen yang tidak didukung secara default.

Layar ini juga memungkinkan anda untuk mengkonfigurasi parameter yang akan diabaikan oleh pemindai yang aktif.
