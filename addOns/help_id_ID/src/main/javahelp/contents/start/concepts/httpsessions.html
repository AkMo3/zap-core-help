Sesi HTTP

Alat ini melacak Sesi HTTP yang ada di Situs tertentu dan memungkinkan pengguna Zaproxy untuk memaksa semua permintaan berada pada sesi tertentu. Pada dasarnya, ini memungkinkan pengguna untuk dengan mudah beralih di antara sesi pengguna di Situs dan membuat Sesi baru tanpa "menghancurkan" yang sudah ada.

Hal ini didasarkan pada konsep dari Sesi Token, yang pesan HTTP parameter (untuk sekarang hanya Cookie) yang memungkinkan sebuah server HTTP untuk menghubungkan pesan permintaan dengan permintaan sebelumnya atau data yang disimpan. Dalam kasus Zaproxy, secara konseptual, token sesi telah dikelompokkan menjadi 2 kategori: token sesi default dan token sesi situs. Default session token adalah orang-orang yang pengguna dapat mengatur dalam Pilihan layar dan token yang, secara default, secara otomatis dianggap token sesi untuk setiap situs (misalnya. phpsessid, jsessionid, dll). Situs token sesi ini adalah satu set token untuk situs tertentu dan biasanya diatur dengan menggunakan popup menu yang tersedia dalam Params Tab.

Alat ini secara otomatis mendeteksi, menggunakan token sesi yang ditentukan atau token sesi default yang terdeteksi secara otomatis, setiap sesi HTTP yang ada dalam komunikasi. Sesi yang terdeteksi ditampilkan pada HTTP Sessions Tab.

Pengguna dapat menggunakan tombol tersedia pada Tab HTTP sesi, menciptakan sesi baru tanpa merusak yang sudah ada, atau dapat memaksa salah satu sesi sebagai 'aktif'. Ketika sesi 'aktif', semua permintaan outbound dikirim ke Situs yang sesuai yang dimodifikasi, sesi token yang dibentuk untuk menyesuaikan sesi aktif. Dengan cara ini, pengguna dapat dengan mudah memaksa beberapa pesanan untuk menjadi 'bagian dari' sesi tertentu dan kemudian beralih dan mengirim pesan pada sesi lain.

Alat HTTP sesi dikonfigurasi menggunakan HTTP sesi pilihan layar.
