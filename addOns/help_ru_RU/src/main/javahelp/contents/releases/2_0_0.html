Release 2.0.0

В этом выпуске были внесены следующие изменения:

Significant changes:

Интегрированные дополнения рынок

ZAP может быть расширен надстройками, которые имеют полный доступ ко всем внутренним ZAP. Любой может написать дополнения и загрузить их на Zap Add-on Marketplace (ОК, так что это проект Google code под названием zap-extensions, но вы получите идею).
Что еще более важно, теперь вы можете просматривать, загружать и устанавливать эти дополнения из ZAP. Большинство надстроек могут быть динамически установлены (и удалены), так что вам даже не потребуется перезагрузка.
Вы можете выбрать, чтобы получать уведомления об обновлениях, и даже автоматически обновляются. И поскольку правила сканирования теперь реализованы в качестве дополнений, вы можете получить последние правила, как только они будут опубликованы.

Замена для "стандартного" Паука

"Старый" Паук показывал свой возраст, поэтому он был полностью переписан, и намного быстрее и более всесторонний, чем старый. Это все еще "традиционный" паук, который анализирует HTML-код для любых ссылок, которые он может найти.

Новый паук "Ajax"

В дополнение к "традиционному" пауку мы добавили Ajax spider, который более эффективен с приложениями, которые интенсивно используют JavaScript. Это использует проект Crawljax, который управляет браузером (используя Selenium), и поэтому может обнаружить любые ссылки, генерируемые приложением, даже те, которые генерируются клиентской стороной.

Web Socket support

Зап теперь поддерживает websockets, поэтому зап можете теперь видеть все через websocket сообщения, отправляемые и получаемые с вашего браузера. Как и в случае с сообщениями на основе HTTP, ZAP также может перехватывать сообщения WebSocket и позволяет изменять их на лету.
Вы также можете пушком сообщения веб-сокетов, а также используя все содержимое фаззинг включенными в зап из проектов, как JBroFuzz и fuzzdb. И, конечно, вы можете легко добавить свои собственные fuzzing файлы.

Вкладка быстрого запуска

The first main tab you will now see is a ‘Quick Start’ tab which allows you to just type in a URL and scan it with one click.
Это идеальная отправная точка для людей, новых для безопасности приложений, но эксперты могут легко удалить его, если они находят его отвлекающим.

Осведомленность о сессии

ZAP теперь осведомленность о сеансе, так что ZAP может распознавать и отслеживать несколько сеансов. Он позволяет создавать новые сеансы, переключаться между ними и применяется ко всем другим компонентам, таким как Spider и активный сканер.

Пользовательские контексты

Теперь вы можете определить любое количество "контекстов" связанных наборов URL - адресов, которые составляют приложение. Затем можно указать все URL-адреса в контексте, например с помощью Spider или активного сканера. Можно также добавить контексты в область и связать другие сведения, такие как сведения проверки подлинности.

Session scope

Область сеанса позволяет указать, какие контексты Вас интересуют в любой момент времени. Вы можете ограничить то, что вы видите на различных вкладках, только URL-адресами в области и предотвратить случайную атаку URL-адресов, не находящихся в области, с помощью защищенного режима.

Different modes

ZAP now supports 3 modes:
  • Сейф, в котором никакие потенциально опасные деятельности не позволили
  • Защищенный, в котором можно выполнять любые действия с URL-адресами в области
  • Стандартный, в котором вы можете сделать что-нибудь для любых URL

Authentication handling

Теперь вы можете связать данные аутентификации с любым контекстом, который позволяет ZAP делать такие вещи, как обнаружение, если и когда вы вышли из системы, и автоматически регистрировать Вас снова. Это особенно полезно при использовании через API в регрессионных тестах безопасности.

More API support

Rest API был значительно расширен, что дает вам гораздо больше доступа к функциональности ZAP обеспечивает.

Fine grained scanning controls

The active scan rules can now be tuned to adjust their strength (the number of attacks they perform) and the threshold at which they report potential issues.

New and improved active and passive scanning rules

We have uploaded the results from running ZAP 2.0.0 against wavsep (the most comprehensive open source evaluation project we are aware of) to the ZAP wiki: https://github.com/zaproxy/zaproxy/wiki/Testing TODO ;)
